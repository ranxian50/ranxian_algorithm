# 一、什么是循环链表
**循环链表（Circular Linked List）**
是**链表的一种变体**，它和普通单链表的唯一核心区别：

> **最后一个节点的 next 不再指向 NULL，而是指向头节点。**

整个链表形成一个**环**。

---

# 二、循环链表的结构
### 1. 节点结构（和单链表完全一样）
```
struct Node {
    数据域 data;
    指针域 next;
};
```

### 2. 循环链表的特点
- 没有 NULL 结尾
- 从**任意一个节点出发，都能遍历整个链表**
- 可以不设头指针，只设**尾指针**，效率更高

---

# 三、循环链表 vs 普通单链表（核心区别）

| 对比项         | 普通单链表                  | 循环链表                     |
|----------------|-----------------------------|------------------------------|
| 最后节点指向   | NULL                        | 头节点                       |
| 遍历结束条件    | p == NULL                   | p == 头节点 或 p->next == 头 |
| 能否从任意点遍历 | 只能从头开始                | 任意节点都能遍历全表         |
| 查找尾节点      | O(n)                        | 若存尾指针则 O(1)            |
| 合并两个链表    | 要遍历找尾                  | 直接改指针，O(1)             |

一句话：
**循环链表 = 带环的单链表。**

---

# 四、循环链表的两种形式
### 1. 带头节点的循环链表（最常用）
- 头节点不存数据
- 最后一个节点 → 头节点

### 2. 不带头节点的循环链表
- 第一个节点就是数据节点
- 最后一个节点 → 第一个数据节点

实际开发、考试、面试**几乎都用带头节点**。

---

# 五、循环链表的基本操作（最详细版）

## 1. 创建空表
```
头节点.next = 头节点;
```
自己指向自己，就是空循环链表。

---

## 2. 判空
```
if (head->next == head)
    空表
```

---

## 3. 遍历（重点！）
普通单链表：
```
while (p != NULL)
```

循环链表：
```
p = head->next;
while (p != head) {
    访问 p->data;
    p = p->next;
}
```

---

## 4. 尾插法（添加到最后）
1. 找到尾节点（p->next == head）
2. newNode->next = head
3. p->next = newNode

**优化：用尾指针 rear**
- 插入尾节点 O(1)
- rear->next = newNode
- newNode->next = head
- rear = newNode

---

## 5. 头插法（添加到最前面）
1. newNode->next = head->next
2. head->next = newNode

---

## 6. 删除节点
假设要删除 p 的后继节点 q：
```
p->next = q->next;
free(q);
```
如果是最后一个节点，依然成立，因为它指向 head。

---

## 7. 合并两个循环链表（面试高频）
给你 A、B 两个循环链表，**只需要改 3 个指针，O(1) 完成合并**：

1. A尾.next = B头.next
2. B尾.next = A头
3. 释放 B 头节点

这是循环链表**最大优势**之一。

---

# 六、循环链表的优点
1. **从任意节点都能遍历全表**
2. **合并链表极快 O(1)**
3. 适合**环形结构业务**（轮询、循环队列）
4. 用尾指针可以让**头尾操作都 O(1)**

---

# 七、循环链表的缺点
1. 容易**死循环**（遍历时一定要判断回到头节点）
2. 不能像数组那样随机访问
3. 比单链表稍微难理解一点

---

# 八、循环链表的经典应用
## 1. 约瑟夫环问题（必考）
- n 个人围成一圈
- 每隔 k 个数淘汰一个
- 求最后存活的人

**用循环链表模拟最直观。**

## 2. 循环队列
用链表实现循环队列，不用判断数组满不满。

## 3. 操作系统时间片轮转调度
进程轮流执行，形成环，循环调度。

## 4. 播放器循环播放列表
播放完最后一首，回到第一首。

# 九、面试常问问题（你直接背）
1. 循环链表和单链表的区别？
   → 最后节点指向头节点，无 NULL，可任意遍历。

2. 循环链表如何判空？
   → head->next == head。

3. 遍历结束条件是什么？
   → p == head。

4. 合并两个循环链表时间复杂度？
   → O(1)。

5. 最适合什么场景？
   → 环形结构、轮询、循环队列、约瑟夫环。
