## 一、什么是双向链表？
双向链表是**链表**的一种，每个节点不只存**数据**和**下一个节点的指针**，还多存一个**指向前一个节点的指针**。

### 结构对比
- **单链表**：`前驱 → 当前 → 后继` 只能往后走
- **双向链表**：`前驱 ← 当前 → 后继` 可以**往前、往后双向遍历**

### 一个节点长这样
```
+------------------+
| 前驱指针 prev     |
+------------------+
|     数据 data     |
+------------------+
| 后继指针 next     |
+------------------+
```

### 整个链表结构
```
NULL ← head ⇄ node1 ⇄ node2 ⇄ ... ⇄ tail → NULL
```

---

## 二、核心特点
1. **可以向前遍历**（单链表做不到）
2. 知道当前节点，就能**直接找到上一个节点**
3. 每个节点**多占一个指针空间**（空间换方便）
4. 插入、删除时**要改更多指针**，逻辑比单链表复杂一点
5. 没有索引，不能随机访问（不能直接 `list[i]`）

---

## 三、双向链表的基本操作（最重点）
下面所有操作我都给你**步骤 + 逻辑图**，保证一看就懂。

### 1. 新建节点
```c
struct Node {
    int data;
    struct Node* prev;
    struct Node* next;
};
```

---

### 2. 在尾部添加节点（尾插）
步骤：
1. 创建新节点
2. 如果链表为空，直接让 head = 新节点
3. 否则找到最后一个节点 tail
4. `tail->next = newNode`
5. `newNode->prev = tail`
6. `newNode->next = NULL`

---

### 3. 在头部添加节点（头插）
1. 新节点 next 指向 head
2. 如果链表不为空：`head->prev = newNode`
3. head 前移到 newNode
4. `newNode->prev = NULL`

---

### 4. 删除指定节点（最常考）
假设要删除节点 `del`：

1. 如果 del 是**头节点**
   - `head = del->next`
   - 如果新头不为空：`head->prev = NULL`

2. 如果 del 是**尾节点**
   - `del->prev->next = NULL`

3. 如果 del 是**中间节点**
   - `del->prev->next = del->next`
   - `del->next->prev = del->prev`

4. 释放 del 内存

> 重点：双向链表删除**不需要从头遍历找前驱**，这是最大优势！

---

### 5. 正向遍历 & 反向遍历
- 正向：从 head 走到 next == NULL
- 反向：从 tail 走到 prev == NULL

单链表**不能反向遍历**。

---

## 四、双向链表 vs 单链表（面试必问）
| 特点              | 单链表                | 双向链表                      |
|-------------------|-----------------------|-------------------------------|
| 指针数量          | 1 个 next             | 2 个 prev + next              |
| 遍历方向          | 只能往后              | 前后都可以                    |
| 删除已知节点      | 必须找前驱，遍历O(n)  | 直接删，O(1)                  |
| 插入难度          | 简单                  | 稍复杂（要改4个指针）|
| 内存占用          | 小                    | 大一点点                     |
| 适用场景          | 简单队列、栈          | 需要频繁前后操作的场景        |

---

## 五、双向链表的优点
1. **删除已知节点极快** O(1)
2. **支持反向遍历**
3. 插入到指定节点**前面/后面**都很方便
4. 很多高级数据结构的基础（如：LRU缓存、链表跳表）

---

## 六、缺点
1. 每个节点多存一个指针，**占更多内存**
2. 插入/删除时要修改更多指针，容易写错
3. 依然**不支持随机访问**，查找还是 O(n)

---

## 七、最常见应用场景
1. **LRU 缓存淘汰算法**（面试高频）
2. 浏览器 **前进/后退历史**
3. 文本编辑器 **撤销/重做**
4. 操作系统 **任务调度链表**
5. Java `LinkedList`、C++ `std::list` 底层都是双向链表

## 八、极简可运行 C 语言示例（方便你直接用）
```c
#include <stdio.h>
#include <stdlib.h>

struct Node {
    int data;
    struct Node* prev;
    struct Node* next;
};

struct Node* createNode(int val) {
    struct Node* newNode = (struct Node*)malloc(sizeof(struct Node));
    newNode->data = val;
    newNode->prev = NULL;
    newNode->next = NULL;
    return newNode;
}

// 尾插
void append(struct Node** head, int val) {
    struct Node* newNode = createNode(val);
    if (*head == NULL) {
        *head = newNode;
        return;
    }
    struct Node* cur = *head;
    while (cur->next != NULL) cur = cur->next;
    cur->next = newNode;
    newNode->prev = cur;
}

// 正向打印
void printForward(struct Node* head) {
    while (head != NULL) {
        printf("%d ", head->data);
        head = head->next;
    }
    printf("\n");
}

int main() {
    struct Node* head = NULL;
    append(&head, 10);
    append(&head, 20);
    append(&head, 30);
    printForward(head);
    return 0;
}
```

## 九、总结（一句话记住）
**双向链表 = 带前驱指针的链表，能往前能往后，删已知节点超快，是很多高级结构的基础。**
