# 数据结构与算法——**串（String）** 超详细全面讲解
串（字符串）是**由零个或多个字符组成的有限序列**，是编程中最常用、最基础的数据结构之一，也是笔试/面试必考内容。

---

## 一、串的基本概念
1. **串的定义**
   - 记为：`S = "a₁a₂…aₙ"`
   - S：串名
   - 引号内：串值
   - n：串长度（n≥0）
   - n=0 时称为**空串**（`""`）

2. **关键区分**
   - **空串**：长度为 0，`""`
   - **空格串**：由空格组成，长度≥1，`"   "`
   - **子串**：串中**任意连续**的字符序列
   - **主串**：包含子串的串
   - **串相等**：长度相同 + 对应位置字符全部相同
   - **模式串**：在匹配中要查找的串

---

## 二、串的逻辑结构与存储结构
串是**线性表的特例**：
- 逻辑上：和线性表一致，是**线性结构**
- 存储上：分为三种

### 1. 顺序存储（定长顺序串）
用**一组地址连续的存储单元**存放字符。

结构（C 风格）：
```c
#define MAXSIZE 256
typedef struct {
    char ch[MAXSIZE];
    int length;
} SString;
```

优点：
- 存取快、随机访问 O(1)
缺点：
- 长度固定，插入、拼接容易溢出

---

### 2. 堆分配存储（动态顺序串）
用**malloc 动态申请内存**，长度可变。

结构：
```c
typedef struct {
    char *ch;
    int length;
} HString;
```

特点：
- 最常用、最实用
- 可动态扩容
- C 语言标准字符串本质就是这种

---

### 3. 块链存储（链式串）
用链表存储，每个结点存多个字符（块）。

优点：插入删除方便
缺点：**效率低、很少实际使用**

---

## 三、串的基本操作
串的操作和普通线性表不同：**以整个串为单位**。

### 常见操作
1. **串赋值**：`StrAssign(&T, chars)`
2. **求串长**：`StrLength(S)`
3. **串比较**：`StrCompare(S, T)`
4. **串连接**：`Concat(&T, S1, S2)`
5. **求子串**：`SubString(&Sub, S, pos, len)`
6. **定位（匹配）**：`Index(S, T)` 找 T 在 S 中第一次出现的位置
7. **串插入**：`StrInsert(&S, pos, T)`
8. **串删除**：`StrDelete(&S, pos, len)`
9. **清空串**：`ClearString(&S)`

## 四、串的模式匹配（核心重点）
**模式匹配 = 在主串 S 中查找模式串 T**
是串最核心算法，面试高频。

---

### 1. 朴素匹配算法（暴力匹配 BF）
**Brute-Force**

思路：
- 主串 i、模式串 j 从 0 开始
- 相等则一起前进
- 不等则 i 回退到 `i-j+1`，j 回退到 0

时间复杂度：
- 最好：O(n+m)
- 最坏：O(n×m)

特点：
- 简单、好写
- 效率低，实际工程不用

---

### 2. KMP 算法（最重要）
**Knuth-Morris-Pratt**
**不回溯主串 i，只回溯模式串 j**，效率大幅提升。

#### 核心思想
利用**已匹配部分的前缀后缀公共部分**，避免重复比较。

#### 关键：next 数组
`next[j]`：当 `S[i] != T[j]` 时，j 应该回退到的位置。
含义：**T[0..j-1] 的最长相等前缀后缀长度**。

#### KMP 步骤
1. 预处理模式串 T，求 next 数组
2. 主串 i 不回退，模式串 j 按 next 回退
3. 匹配成功返回位置，失败返回 -1

时间复杂度：**O(n + m)**

---

### 3. 扩展：nextval 数组（优化 KMP）
对 next 进一步优化，避免重复比较相同字符。
面试常问：**KMP + next + nextval**。
