# 一、树是什么？（本质定义）
**树是一种**非线性、分层、一对多**的数据结构。**

特点：
- 由**节点（Node）**和**边（Edge）**组成
- 有且只有一个**根节点（Root）**
- 除根外，每个节点有且只有一个**父节点**
- 无环（这是树和图最核心区别）
- 天然**递归结构**

一句话：**树 = 有根、无环、连通的分层结构**。

---

# 二、树的基本术语（必须背熟）
1. **根节点（Root）**：最顶层节点，没有父节点
2. **父节点（Parent）**：一个节点的直接上层节点
3. **子节点（Child）**：一个节点的直接下层节点
4. **叶子节点（Leaf）**：没有子节点的节点
5. **边（Edge）**：连接两个节点的线
6. **路径（Path）**：从一个节点到另一个节点经过的边
7. **深度（Depth）**：从**根**到该节点的层数（根一般为 0 或 1）
8. **高度（Height）**：从该节点到**最深叶子**的最长路径
9. **子树（Subtree）**：以某个节点为根的树
10. **度（Degree）**：一个节点拥有的子节点数量

# 三、最核心：二叉树（Binary Tree）
**二叉树 = 每个节点最多 2 个子节点（左、右）**

这是算法里**最重要、考最多**的树结构。

## 1. 二叉树的性质
- 第 i 层最多有 **2ⁱ** 个节点（根 i=0）
- 高度 h 的二叉树最多 **2ʰ⁺¹ - 1** 个节点
- n 个节点的二叉树高度至少为 **log₂(n+1) - 1**
- 叶子数 = 度为 2 的节点数 + 1

---

## 2. 特殊二叉树
### （1）满二叉树
- 所有层都填满
- 叶子全部在最底层
- 节点总数：2ʰ⁺¹ - 1

### （2）完全二叉树
- 除最后一层外都填满
- 最后一层靠左排列
- **堆（Heap）就是完全二叉树**

### （3）二叉搜索树 BST
**左子树 < 根 < 右子树**
- 中序遍历 = 升序序列
- 查找、插入、删除平均 O(log n)
- 最坏退化成链表 O(n)

### （4）平衡二叉树（AVL）
- 左右子树高度差 ≤ 1
- 保证所有操作稳定 O(log n)

### （5）红黑树
- 弱平衡二叉搜索树
- Java TreeMap、C++ STL::map、Linux内核都用它

# 四、二叉树遍历（必考核心）
遍历 = 按某种规则访问所有节点。

## 1. 深度优先 DFS（递归最常用）
### （1）前序遍历
**根 → 左 → 右**
```
preOrder(node) {
    if(node==null) return;
    visit(node);
    preOrder(left);
    preOrder(right);
}
```

### （2）中序遍历
**左 → 根 → 右**
- BST 中序 = 有序

### （3）后序遍历
**左 → 右 → 根**
- 常用于删除树、计算子树信息

---

## 2. 广度优先 BFS（层序遍历）
**一层一层遍历**
用队列实现：
```
levelOrder(root) {
    queue.add(root);
    while(!queue.isEmpty()) {
        node = queue.poll();
        visit(node);
        if(left) queue.add(left);
        if(right) queue.add(right);
    }
}
```

# 五、重要高级树结构（面试/工程必知）

## 1. 堆（Heap）
- **完全二叉树**
- 大顶堆：父 ≥ 子
- 小顶堆：父 ≤ 子
用途：
- 优先队列
- 堆排序 O(n log n)
- TopK 问题

## 2. 字典树 Trie
多叉树，用于**字符串前缀匹配**
- 每个节点存字符
- 从根到叶子构成单词
用途：搜索引擎提示、自动补全、IP 路由

## 3. 并查集（不相交集合）
树结构用来**管理分组与连通性**
核心操作：
- find（找根）
- union（合并）
用途：朋友圈、岛屿数量、最小生成树 Kruskal

## 4. B 树 / B+ 树
**平衡多路查找树**
- MySQL 索引核心就是 B+ 树
- 磁盘友好、减少 I/O

# 六、树的常见存储方式
## 1. 链式存储（最常用）
```
class TreeNode {
    int val;
    TreeNode left;
    TreeNode right;
}
```

## 2. 数组存储（完全二叉树）
- 根：i=0 或 1
- 左孩子：2i+1
- 右孩子：2i+2
- 父：(i-1)/2

# 八、树的核心思想（掌握=会做80%题）
1. **递归是天然写法**
   树 = 根 + 左子树 + 右子树
2. **DFS 用栈 / 递归**
3. **BFS 用队列**
4. **子问题思想**
   求整棵树 → 先求左、右子树
5. **回溯思想**
   路径问题必用回溯

# 九、树 vs 链表 vs 图（快速区分）
- 链表：**线性、1对1**
- 树：**分层、1对多、无环**
- 图：**多对多、可有环**
