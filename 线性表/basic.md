# 一、什么是线性表？
**线性表 = 具有相同数据类型的 n 个数据元素的有限序列。**

一句话理解：
**像排队一样，从头到尾一条直线，每个元素只有一个前驱、一个后继。**

特点：
1. **有序**：元素之间有先后顺序。
2. **有限**：元素个数有限。
3. **同类型**：所有元素类型一样（int、char、结构体等）。
4. **唯一前驱、唯一后继**：
   - 除了第一个，每个都有**前驱**
   - 除了最后一个，每个都有**后继**

典型例子：
数组、排队、学号列表、字母表。

---

# 二、线性表的两种核心实现
线性表只是**逻辑结构**，真正在代码里有两种物理实现：

1. **顺序表（顺序存储）**
2. **链表（链式存储）**

这是数据结构最核心的考点、面试点、工程区分点。

---

# 三、顺序表（数组实现）
## 1. 定义
用**一段连续的内存空间**依次存储线性表元素。
本质就是：**数组**。

## 2. 结构
内存地址连续，像一排连在一起的房间。

## 3. 核心操作（时间复杂度）
- **访问第 i 个元素**：O(1)
  直接下标定位，超快。
- **插入**：
  头部/中间插入 → O(n)（要移动后面所有元素）
  尾部插入 → O(1)
- **删除**：
  头部/中间删除 → O(n)
  尾部删除 → O(1)
- **查找**：
  无序：O(n)
  有序：可二分 O(log n)

## 4. 优点
- 随机访问快（下标直接找）
- 内存连续，缓存友好，速度极快
- 实现最简单

## 5. 缺点
- 插入删除慢（要搬数据）
- 长度固定，扩容麻烦（需要重新开辟空间复制）
- 空间利用率可能不高（预分配太大浪费）

## 6. 适用场景
- 读多写少
- 元素个数稳定
- 需要频繁查找、遍历
- 比如：排行榜、固定配置表、静态数组

# 四、链表（链式存储）
用**不连续的节点**通过指针/引用连起来。

每个节点包含：
1. **数据域**：存值
2. **指针域**：指向下一个节点

## 链表分 4 种（考试/面试必考）
1. **单链表**
2. **双链表**
3. **循环单链表**
4. **循环双链表**

---

## 1）单链表
- 每个节点只有一个 next 指针
- 只能从前往后遍历
- 无法回头

操作：
- 访问：O(n)（必须从头遍历）
- 插入/删除：已知节点时 O(1)，找节点 O(n)
- 不支持随机访问

---

## 2）双链表
- 每个节点有 **prev + next**
- 可向前、可向后遍历
- 插入删除更方便

优点：
- 找前驱不用遍历
- 插入删除效率比单链表高

---

## 3）循环单链表
- 最后一个节点的 next 指向头节点
- 形成环
- 适合环形结构（轮询、循环队列）

---

## 4）循环双链表
- 最实用、最强大的链表结构
- 头 prev 指向尾，尾 next 指向头
- 任意位置都能方便遍历全表

---

# 链表通用优点
- **插入删除极快**（只要找到位置，改指针就行）
- **动态大小**，不用预先分配空间
- 不会浪费内存

# 链表通用缺点
- **不能随机访问**，找元素必须遍历
- 每个节点多存指针，额外开销
- 对CPU缓存不友好（内存跳来跳去）

---

# 五、顺序表 vs 链表（终极对比表）
| 对比项         | 顺序表                | 链表                    |
|----------------|-----------------------|-------------------------|
| 存储结构       | 连续内存              | 不连续，通过指针连接    |
| 随机访问       | 支持 O(1)             | 不支持 O(n)             |
| 插入删除       | 中间慢 O(n)           | 已知位置快 O(1)         |
| 内存分配       | 静态/动态扩容         | 动态分配，随用随申请    |
| 空间利用率     | 可能浪费              | 无浪费，但有指针开销    |
| 查找速度       | 有序可二分 O(log n)   | 只能遍历 O(n)           |
| 适用场景       | 读多、少改、固定长度  | 写多、频繁增删、长度不定 |

一句话总结：
**读多用顺序，写多用链表。**

# 七、线性表的实际应用（你每天都在用）
- **数组**：顺序表
- **ArrayList / Vector**：动态顺序表
- **LinkedList**：双链表
- **队列、栈**：底层都是线性表
- **LRU 缓存**：用哈希+双链表
- **操作系统进程管理**：链表
- **浏览器前进后退**：双链表
- **Redis 列表**：quicklist（链表+压缩数组）
