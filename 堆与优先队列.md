# 一、先一句话搞懂：它们是什么关系？
- **优先队列 = 一种“按优先级出队”的队列**
- **堆 = 实现优先队列最高效的数据结构**

关系就像：
> **数组 / 链表 ≈ 栈 / 队列**
> **堆 ≈ 优先队列**

---

# 二、普通队列 vs 优先队列
## 1. 普通队列（Queue）
- 先进先出 FIFO
- 谁先来谁先走

## 2. 优先队列（Priority Queue）
- **不是按进入顺序，而是按优先级出队**
- 优先级最高的永远先出来
- 插入随便插，但取的时候永远拿最大/最小

**生活例子：**
- 医院急诊：重伤优先，不管谁先来
- 任务调度：重要任务优先执行
- 游戏：伤害最高的技能优先触发

---

# 三、堆（Heap）到底是什么？
堆是一棵**完全二叉树**，且满足：

## 大顶堆（最大堆）
- 父节点 ≥ 子节点
- 堆顶是**最大值**

## 小顶堆（最小堆）
- 父节点 ≤ 子节点
- 堆顶是**最小值**

### 关键特点
1. **完全二叉树**
   除最后一层，其他层都满；最后一层靠左排列
2. **可以用数组存储！**（这是堆最牛的地方）
3. 插入、删除都是 **O(log n)**

---

# 四、堆为什么能用数组存？（超级重要）
因为是完全二叉树，我们可以直接用下标映射：

假设数组从 **下标 1 开始存**：

- 节点 i 的**左孩子**：`2*i`
- 节点 i 的**右孩子**：`2*i + 1`
- 节点 i 的**父节点**：`i/2`（向下取整）

如果从 0 开始：
- 左：`2*i + 1`
- 右：`2*i + 2`
- 父：`(i-1)/2`

**堆 = 数组 + 完全二叉树结构 + 堆性质**

---

# 五、堆的两个核心操作（一定要懂）
## 1. 上浮（shift up / swim）
用于**插入**后，维护堆结构。

步骤：
1. 新元素插到数组最后
2. 和父节点比较
3. 如果比父大（大顶堆），就交换
4. 直到不能交换或到堆顶

## 2. 下沉（shift down / sink）
用于**删除堆顶**后，维护堆结构。

步骤：
1. 堆顶和最后一个元素交换
2. 删除最后一个元素
3. 堆顶开始下沉：和左右孩子较大的比较
4. 比孩子小就交换
5. 直到不能下沉

**插入、删除都是 O(log n)**
这就是优先队列高效的原因。

---

# 六、堆排序（HeapSort）
思路超简单：
1. 把数组建成堆（O(n)）
2. 每次把堆顶放到最后
3. 重新下沉
4. 最后得到**有序数组**

时间复杂度：
**O(n log n)**
不稳定排序。

---

# 七、优先队列的实际用途（面试高频）
- 任务调度
- 定时器（最小堆，最早触发的先执行）
- 哈夫曼编码
- Dijkstra 最短路径
- 合并 K 个有序链表
- Top K 问题（第 K 大、前 K 大）

---

# 八、最经典题型：Top K 问题
给你一个数组，求**第 K 大元素** / **前 K 大元素**

最优解法：**小顶堆**
1. 维护一个大小为 K 的小顶堆
2. 比堆顶大就替换
3. 最后堆顶就是第 K 大
4. 堆里就是前 K 大

时间：**O(n log K)**

---

# 九、C++ 里怎么用？（你最需要的）
C++ STL 提供：
```cpp
#include <queue>
```

## 1. 默认大顶堆
```cpp
priority_queue<int> pq;
```
- 堆顶是最大的

## 2. 小顶堆（常用！）
```cpp
priority_queue<int, vector<int>, greater<int>> pq;
```

## 常用方法
```cpp
pq.push(x);    // 插入 O(log n)
pq.top();      // 取堆顶 O(1)
pq.pop();      // 删除堆顶 O(log n)
pq.size();
pq.empty();
```

# 十、最简单总结（背会就能面试）
1. **优先队列 = 按优先级出队**
2. **堆 = 实现优先队列的完全二叉树**
3. 堆分**大顶堆、小顶堆**
4. 堆用**数组存储**，靠下标找父子
5. 核心操作：**上浮、下沉**，O(log n)
6. C++：`priority_queue`
7. 最常考：**Top K、堆排序、Dijkstra**
