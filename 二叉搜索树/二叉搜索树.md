# 二叉搜索树

**二叉搜索树**(Binary Search Tree, BST)，又称**二叉排序树**,是一种兼具"有序性"与"高效操作"的树形数据结构。它在查找、插入和删除操作上表现优异，是理解更复杂数据结构 (如 AVL 树、红黑树) 的基础。

## 🌳 核心概念与性质

二叉搜索树 (BST) 或者是一棵空树，或者是满足以下性质的二叉树:

1. **左小右大**: 若节点的左子树不为空，则左子树上所有节点的值都小于 (或等于) 该节点的值。
2. **右大左小**: 若节点的右子树不为空，则右子树上所有节点的值都大于 (或等于) 该节点的值。
3. **递归定义**: 节点的左、右子树也分别是二叉搜索树。

### 关键特性

- **有序性**: 对二叉搜索树进行中序遍历 (左子树 -> 根节点 -> 右子树),得到的结果是一个升序排列的序列。这是 BST 最核心的特性，也是其"排序树"名称的由来。
- **唯一性**: 通常情况下，BST 中不允许存在键值相等的节点。但这一规则并非绝对，具体取决于应用场景。例如，`std::set` 不允许重复，而 `std::multiset` 则允许。

## ⚙️ 核心操作

BST 的所有操作都基于其"左小右大"的性质，从根节点开始，通过与当前节点值的比较来决定下一步是向左子树还是右子树探索。

### 查找 (Search)

查找过程类似于二分查找，效率很高。

1. 从根节点开始，将目标值与当前节点的值进行比较。
2. 如果相等，则查找成功。
3. 如果目标值小于当前节点的值，则继续在左子树中查找。
4. 如果目标值大于当前节点的值，则继续在右子树中查找。
5. 如果最终到达空节点，则说明查找失败。

### 插入 (Insert)

插入的节点总是会成为树中的一个新叶子节点。

1. 从根节点开始，按照查找的逻辑找到新节点应该插入的位置。
2. 当到达一个空位置时，将新节点插入此处。

### 删除 (Delete)

删除操作是 BST 中最复杂的，需要分三种情况处理:

1. **删除叶子节点**: 如果待删除的节点没有子节点，直接将其删除即可。
2. **删除只有一个子节点的节点**: 用其唯一的子节点替换待删除的节点，然后删除原节点。
3. **删除有两个子节点的节点**: 这是最复杂的情况。通常的做法是:
   - 找到待删除节点的中序后继 (即其右子树中的最小值节点) 或中序前驱 (即其左子树中的最大值节点)。
   - 用后继 (或前驱) 节点的值覆盖待删除节点的值。
   - 然后，在右子树 (或左子树) 中递归地删除那个后继 (或前驱) 节点。

## 📊 性能分析

BST 操作的时间复杂度主要取决于树的高度 $h$。

| 场景 | 树的形态 | 高度 (h) | 时间复杂度 |
|------|----------|----------|------------|
| 理想情况 | 完全二叉树 (接近平衡) | $\log_2 N$ | $O(\log N)$ |
| 最差情况 | 单支树 (退化为链表) | $N$ | $O(N)$ |

- **理想情况**: 当节点随机插入时，树的结构会比较平衡，高度接近 $\log_2 N$,操作效率很高。
- **最差情况**: 当数据按有序序列 (如 1, 2, 3, 4, 5) 插入时，树会退化成一个链表，所有操作的效率都降为 $O(N)$。

### 与二分查找的对比

| 特性 | 二分查找 | 二叉搜索树 |
|------|----------|------------|
| 存储结构 | 必须是支持随机访问的有序数组 | 链式存储，无需连续空间 |
| 插入/删除 | 效率低，需移动大量元素 $O(N)$ | 效率高，仅需修改指针 $O(\log N)$ |
| 适用场景 | 静态数据 (极少增删) | 动态数据 (频繁增删查改) |

## 🔄 树的遍历

遍历是指按照特定顺序访问树中的每一个节点。除了递归实现，掌握非递归 (迭代) 实现也至关重要，可以避免在树过深时发生栈溢出。

### 前序遍历 (Pre-order)

- **顺序**: 根节点 -> 左子树 -> 右子树
- **非递归实现**: 使用一个栈。先将根节点入栈，然后循环执行：弹出栈顶节点并访问，然后先将其右子节点入栈，再将其左子节点入栈。

### 中序遍历 (In-order)

- **顺序**: 左子树 -> 根节点 -> 右子树
- **非递归实现**: 使用一个栈。从根节点开始，沿着左子树路径将所有节点入栈，直到最左侧的叶子节点。然后弹出栈顶节点并访问，再将当前节点指向其右子树，重复此过程。

### 后序遍历 (Post-order)

- **顺序**: 左子树 -> 右子树 -> 根节点
- **非递归实现 (双栈法)**: 使用两个栈。将根节点压入栈 1。循环执行：从栈 1 弹出节点并压入栈 2，然后先将其左子节点压入栈 1，再将其右子节点压入栈 1。最后，依次弹出并访问栈 2 中的节点。

### 层序遍历 (Level-order)

- **顺序**: 从上到下，从左到右逐层访问。
- **非递归实现**: 使用一个队列。将根节点入队，然后循环执行：弹出队首节点并访问，然后将其左、右子节点 (如果存在) 依次入队。

## 📝 代码实现示例 (C++)

以下是一个简化的 C++ 模板实现，展示了 BST 的核心结构。

```cpp
#include <iostream>
#include <stack>
#include <queue>

template <typename T>
class BinarySearchTree {
private:
    struct Node {
        T data;
        Node* left;
        Node* right;
        Node(T val) : data(val), left(nullptr), right(nullptr) {}
    };
    Node* root;

    // 递归插入辅助函数
    Node* insertRecursive(Node* node, T val) {
        if (node == nullptr) {
            return new Node(val);
        }
        if (val < node->data) {
            node->left = insertRecursive(node->left, val);
        } else if (val > node->data) {
            node->right = insertRecursive(node->right, val);
        }
        // 如果值相等，这里选择不插入
        return node;
    }

    // 递归查找辅助函数
    bool searchRecursive(Node* node, T val) {
        if (node == nullptr) return false;
        if (val == node->data) return true;
        if (val < node->data) return searchRecursive(node->left, val);
        return searchRecursive(node->right, val);
    }

    // 非递归中序遍历
    void inorderTraversalIterative() {
        std::stack<Node*> s;
        Node* current = root;
        while (current != nullptr || !s.empty()) {
            while (current != nullptr) {
                s.push(current);
                current = current->left;
            }
            current = s.top();
            s.pop();
            std::cout << current->data << " ";
            current = current->right;
        }
        std::cout << std::endl;
    }

public:
    BinarySearchTree() : root(nullptr) {}

    void insert(T val) {
        root = insertRecursive(root, val);
    }

    bool search(T val) {
        return searchRecursive(root, val);
    }

    void inorder() {
        inorderTraversalIterative();
    }
};

// 使用示例
int main() {
    BinarySearchTree<int> bst;
    bst.insert(50);
    bst.insert(30);
    bst.insert(70);
    bst.insert(20);
    bst.insert(40);

    std::cout << "中序遍历：";
    bst.inorder(); // 输出：20 30 40 50 70

    std::cout << "查找 40: " << (bst.search(40) ? "找到" : "未找到") << std::endl;
    std::cout << "查找 60: " << (bst.search(60) ? "找到" : "未找到") << std::endl;

    return 0;
}
```

## 🚀 应用场景与优化

### 应用场景

BST 广泛应用于需要高效查找和动态增删的场景，如:
- 实现字典
- 符号表
- 数据库索引
- 文件系统

### 优化方向

为了避免 BST 在最差情况下退化为链表，计算机科学家提出了**自平衡二叉搜索树**,如:
- **AVL 树**: 通过严格的平衡条件确保树的高度
- **红黑树**: 通过颜色标记和旋转操作维持近似平衡

这些数据结构通过在插入和删除操作后自动调整树的结构，确保树的高度始终保持在 $O(\log N)$ 级别，从而保证了操作的高效和稳定。
