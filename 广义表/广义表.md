# 广义表（Generalized List）全面详解
广义表是**线性表的高阶推广**，是允许元素既可以是不可再分的原子（Atom），也可以是另一个广义表（子表 SubList）的非线性数据结构，核心是**递归嵌套**，能灵活表示层次化、共享化的复杂数据关系。

---

## 一、基本定义与逻辑结构
### 1. 形式化定义
广义表是 **n（n≥0）** 个元素的有限序列，记作：
$$LS=(a_1,a_2,a_3,\dots,a_n)$$
- **LS**：广义表名称（大写）
- **aᵢ**：表元素（小写表原子，大写表子表），可原子/可子表
- **n**：**表长度**（最外层元素个数），n=0 为**空表**，记作 `()` 或 `LS=()`

### 2. 核心术语（必考）
#### （1）表头（Head）
非空广义表的 **第一个元素**（可以是原子或子表）
> 例：`L=(a,(b,c),d)` → `head(L)=a`

#### （2）表尾（Tail）
非空广义表 **除表头外，其余所有元素组成的子表**（**永远是表**）
> 例：`L=(a,(b,c),d)` → `tail(L)=((b,c),d)`

#### （3）长度（Length）
**最外层**元素个数（不计嵌套）
- `A=()` → 长度 0
- `B=(a,b)` → 长度 2
- `C=(a,(b,c),d)` → 长度 3

#### （4）深度（Depth）
**括号嵌套的最大层数**（递归层数）
- 原子深度 = **0**
- 空表 `()` 深度 = **1**
- 规则：`depth(LS)=1+max{depth(aᵢ)}`

**示例计算**：
- `D=((), (a,(b)), c)`
  - `depth(())=1`
  - `depth((a,(b)))=1+max(0,1+0)=2`
  - `depth(c)=0`
  - `depth(D)=1+max(1,2,0)=3`

### 3. 四大特性
1. **层次性**：多层嵌套，形成树状层次
2. **递归性**：可包含自身（如 `E=(a,E)`）
3. **共享性**：多个表可共用同一子表（节省空间）
4. **非线性**：元素间一对多关系，属非线性结构

### 4. 典型示例
- `A=()`：空表，长0，深1
- `B=(x,y,z)`：纯原子表，长3，深1
- `C=(a,(b,c))`：含子表，长2，深2
- `D=(C,((d),e))`：多重嵌套，长2，深4
- `E=(a,E)`：递归表，长2，深∞
- `F=(A,B,C)`：共享表（含A、B、C）

## 二、存储结构（链式为主）
广义表无法用纯顺序存储，**主流采用链式结构**，分两种：

### 1. 头尾链表（最常用）
#### （1）结点设计
两类结点（用 **tag 标志位**区分）：
- **原子结点（tag=0）**：`tag + 数据域 (atom)`
- **表结点（tag=1）**：`tag + 表头指针 (hp) + 表尾指针 (tp)`

**C 语言定义**：
```c
typedef enum { ATOM, LIST } ElemTag; // ATOM=0原子, LIST=1子表

typedef struct GLNode {
    ElemTag tag;
    union {
        // 原子结点的值域
        int atom;
        // 表结点的hp(表头)、tp(表尾)
        struct { struct GLNode *hp, *tp; } htp;
    } atom_htp;
} GLNode, *GList;
```


#### （2）存储示例
`LS=(a,(b,c))` 存储结构：
```
LS → [tag=1, hp→a结点, tp→表结点]
          ↓
        [tag=0, atom='a']
                    ↓
                [tag=1, hp→b结点, tp→c结点]
                            ↓
                        [tag=0, atom='b']
                                    ↓
                                [tag=1, hp→c结点, tp=NULL]
                                            ↓
                                        [tag=0, atom='c']
```

### 2. 同层结点链（扩展链表）
- 同一层元素用 **tp 指针**串成单链表
- 表结点用 `hp` 指向子表，原子结点用 `data` 存值
- 结构更紧凑、遍历更直观

## 三、基本操作（递归实现）
广义表操作**几乎全靠递归**（自身嵌套定义）。

### 1. 求长度
- 空表：0
- 非空：1 + 表尾长度
```c
int GListLength(GList L) {
    if (!L) return 0; // 空表
    return 1 + GListLength(L->atom_htp.htp.tp);
}
```

### 2. 求深度
- 原子：0
- 空表：1
- 表：1 + max(表头深度, 表尾深度)
```c
int GListDepth(GList L) {
    if (!L) return 1; // 空表
    if (L->tag == ATOM) return 0; // 原子
    int dH = GListDepth(L->atom_htp.htp.hp);
    int dT = GListDepth(L->atom_htp.htp.tp);
    return 1 + (dH > dT ? dH : dT);
}
```

### 3. 取表头 / 表尾
```c
// 取表头
GList GetHead(GList L) {
    if (!L || L->tag == ATOM) return NULL;
    return L->atom_htp.htp.hp;
}

// 取表尾
GList GetTail(GList L) {
    if (!L || L->tag == ATOM) return NULL;
    return L->atom_htp.htp.tp;
}
```

### 4. 遍历 / 打印
```c
void PrintGList(GList L) {
    if (!L) { printf("()"); return; }
    if (L->tag == ATOM) { printf("%d", L->atom_htp.atom); return; }
    printf("(");
    GList p = L;
    while (p) {
        PrintGList(p->atom_htp.htp.hp);
        p = p->atom_htp.htp.tp;
        if (p) printf(",");
    }
    printf(")");
}
```

### 5. 复制（递归深拷贝）
```c
int CopyGList(GList &T, GList L) {
    if (!L) { T = NULL; return 1; }
    T = (GList)malloc(sizeof(GLNode));
    T->tag = L->tag;
    if (L->tag == ATOM)
        T->atom_htp.atom = L->atom_htp.atom;
    else {
        CopyGList(T->atom_htp.htp.hp, L->atom_htp.htp.hp);
        CopyGList(T->atom_htp.htp.tp, L->atom_htp.htp.tp);
    }
    return 1;
}
```

## 四、广义表 vs 线性表 / 树 / 图
| 特性 | 线性表 | 广义表 | 树 | 图 |
| :--- | :--- | :--- | :--- | :--- |
| **元素类型** | 仅原子 | 原子+子表 | 结点+子树 | 顶点+边 |
| **关系** | 一对一 | 一对多（嵌套） | 严格一对多 | 多对多 |
| **递归** | 不支持 | 支持 | 天然递归 | 可环 |
| **共享** | 不支持 | 支持 | 不支持 | 支持 |
| **结构** | 线性 | 半线性/非线性 | 层次 | 网状 |

**关键区别**：
- 广义表 ≠ 树：**表尾是线性序列**、**支持共享/递归**
- 广义表 ≠ 线性表：**允许子表嵌套**、**非线性**

## 五、应用场景
1. **LISP 语言**：程序与数据统一用广义表表示
2. **多元多项式表示**：`(x, (y, 2), (z, 3))` 表示 `x+2y+3z`
3. **树形结构表示**：`(A(B,C(D,E)))` 表示树
4. **文件系统/目录结构**：文件夹（子表）+ 文件（原子）
5. **AI 知识表示、符号计算**

---

## 六、易错点总结
1. **表尾一定是表**，绝不是原子
2. **空表长度=0、深度=1**
3. **递归表深度无穷大**
4. **共享表需避免重复释放**（内存管理）
5. 广义表是**非线性结构**（常考）
